18   * <p>Input adapters take information from outside of an application and
19   * provide it in the form of a stream or streams to the application. 
20   * Any implementation of this class <b>must</b> extend class InputAdapter.
21   * Input adapters should call sendOutput(int, Tuple) to provide tuples 
22   * to the application.</p>
23   * 
24   * <p>For in-depth information on implementing a custom adapter, please
25   * see "Developing StreamBase Embedded Adapters" in the StreamBase documentation.</p>

27  public class SampleAdapter extends InputAdapter implements Parameterizable, Runnable {

34      /**
35       * The constructor is called when the Adapter instance is created, but before the Adapter 
36       * is connected to the StreamBase application. We recommended that you set the initial input 
37       * port and output port count in the constructor by calling setPortHints(inPortCount, outPortCount). 
38       * The default is no input ports or output ports. The constructor may also set default values for 
39       * adapter parameters. These values will be displayed in StreamBase Studio when a new adapter is 
40       * dragged to the canvas, and serve as the default values for omitted optional parameters.
41       */
42      public SampleAdapter() {
43          super();
44          logger = getLogger();
45          setPortHints(0, 1);
46          setDisplayName(displayName);
47          setShortDisplayName(this.getClass().getSimpleName());
48      }

54       * The message associated with the thrown exceptions will be displayed in StreamBase Studio during authoring,
55       * or printed on the console by sbd. Input adapters should set the schema of each output
56       * port by calling the setOutputSchema(portNum, schema) method for each output port. 
57       * If the adapter needs to change the number of input ports based on parameter values,
58       * it should call requireInputPortCount(portCount) at this point. 
59       */
60      public void typecheck() throws TypecheckException {
61          // TODO Make sure all properties have valid values.
62          // Sets the output schema for the given output port (port #'s are zero based).
63          setOutputSchema(0, schema0);
64      }

66      /**
67       * Initialize the adapter. If typecheck succeeds, the init method is called before
68       * the StreamBase application is started. Note that your adapter is not required to
69       * define the init method, unless you need to register a runnable or perform
70       * initialization of a resource such as, for example, a JDBC pool.
71       */
72      public void init() throws StreamBaseException {
73          super.init();
74  
75          // Register the object so it will be run as a thread managed by StreamBase.
76          registerRunnable(this, true);
77      }

79      /**
80       * Shutdown adapter by cleaning up any resources used (e.g. close an output file

85      public void shutdown() {
86  
87      }

89      /**
90       * Main thread of the adapter. At this point, the application begins to run.
91       * StreamBase will start threads for any managed runnables registered by
92       * earlier calls to registerRunnable. Input adapters can call sendOutput(port, tuple) 
93       * at any time to output a tuple to the specified port.
94       */
95      public void run() {
96          // fill the tuple and output the tuple
97          fillAndOutputTuple();
98      }
99  
100     private void fillAndOutputTuple() {
101         if (logger.isInfoEnabled()) {
102             logger.info("fill the tuple and output the tuple...");
103         }
104         try {
105             while (shouldRun()) {
106                 int portCount = getOutputPortCount();
107                 for (int i = 0; i < portCount; i++) {
108                     // TODO Get output schema and create tuple from the schema
109                     Tuple tuple = getOutputSchema(i).createTuple();
110 
111                     // TODO Accept input from other resources
112 
113                     // TODO set the tuple fields as desired based on accepted input
114 
115                     try {
116                         // Output the tuple
117 
118                         sendOutput(i, tuple);
119                     } catch (StreamBaseException e) {
120                         System.err.println("Exception sending output to port"
121                                 + portCount);
122                         System.err.println(e);
123                         return;
124                     }
125                     try {
126                         // TODO this sleep is only here to throttle this simulated data source.
127                         // Real data sources are likely not to need any throttling as the StreamBase
128                         // Server is almost always capable of higher input rates than remote data
129                         // sources can output, and buffering is managed by the Server if needed.
130                         logger.info("Sleeping 250ms");
131                         Thread.sleep(250);
132                     } catch (InterruptedException e) {
133                     }
134                 }
135             }
136         } finally {
137             try {
138                 // TODO Be sure to close the resource
139             } catch (Exception e) {
140             }
141         }
142     }
